OMINAS Examples

Introduction

OMINAS example scripts demonstrate various ways to use the software, and may form the basis for your own OMINAS scripts. The examples are also useful for testing an installation. The OVERRIDE example is a good way to test the basic installation, as it requires no translators. The PG example is a good way to test your ICY installation, assuming you're using the basic demo coniguration, which uses the NAIF/SPICE translator.

To navigate using the sidebar, click on a directory in the upper frame and then a program in the lower frame. When you click on a directory, the lower frame will display every .pro file in that directory. Clicking on the .pro file will take you to that program's documentation.

Running the Examples

Before running the examples, make sure you have configured the demo package using the installer. Examples named *-batch.pro are IDL batch files, and may be run from the IDL command line as:


IDL> @<name>_example-batch.pro

or from the shell prompt as:.


% ominas <name>_example-batch.pro

Examples named *.pro are $MAIN$ programs that may be run as:


IDL> .run <name>_example.pro

or from the shell prompt as:.


% ominas -e ".run <name>_example.pro"

The two types of scripts have their own avantages and disadvantages. Batch files are more easily executed from the shell prompt, and may be pasted line-by-line into IDL from a text editor. $MAIN$ programs are witten more like programs files and do not require line extensions in blocks of code.
